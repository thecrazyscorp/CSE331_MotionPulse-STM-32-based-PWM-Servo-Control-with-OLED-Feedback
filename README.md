> An autonomous fan speed controller using a continuous-rotation servo motor driven by STM32 hardware PWM. The system cycles through 5 speed stages automatically and displays real-time status — mode, direction, PWM pulse, and next-stage countdown — on a 0.96" SSD1306 OLED over I2C.

---

## 📌 Overview

**MotionPulse** is an embedded systems project built on the STM32F103C8T6 (Blue Pill). It drives a **continuous-rotation servo motor** through 5 pre-defined speed stages using hardware PWM generated by **TIM2 Channel 2**. The system autonomously ramps up and then ramps back down in a ping-pong loop, holding each stage for **10 seconds**.

A **SSD1306 0.96" OLED display** connected over **I2C1 at 400 kHz (Fast Mode)** provides a live dashboard showing the current speed mode, direction, PWM pulse width, and a countdown to the next stage transition.

## ⚙️ How It Works

### PWM Generation — TIM2 Channel 2

The STM32's **TIM2** timer drives the servo signal on **PA1 (TIM2_CH2)**. The timer is configured with:

```c
htim2.Init.Prescaler = 7;       // 8 MHz HSI / (7+1) = 1 MHz timer clock
htim2.Init.Period    = 19999;   // 1 MHz / 20000 = 50 Hz (20 ms period)
```

This gives a **1 µs resolution** per tick. Pulse widths map directly to motor speed and direction:

| Pulse Width | Stage | Speed | Direction |
|---|---|---|---|
| 1500 µs | STOP | 0% | Stopped |
| 1700 µs | LOW | 25% | Clockwise |
| 1900 µs | MED | 50% | Clockwise |
| 2100 µs | HIGH | 75% | Clockwise |
| 2500 µs | MAX | 100% | Clockwise |

### Stage Sequencer

The system uses a **ping-pong state machine** — automatically stepping up through all 5 stages, then reversing back down, looping indefinitely. Each stage is held for **10 seconds** (`STAGE_HOLD_TIME_MS = 10000`):

```
STOP → LOW → MED → HIGH → MAX → HIGH → MED → LOW → STOP → ...
```

```c
// Direction flag drives the bounce logic
if (direction == 1) {
    if (currentStage < STAGE_COUNT - 1) currentStage++;
    else { direction = 0; currentStage--; }
} else {
    if (currentStage > 0) currentStage--;
    else { direction = 1; currentStage++; }
}
```

### OLED Dashboard — I2C1 Fast Mode (400 kHz)

The display refreshes every **500 ms** and shows a 5-line live dashboard:

```
  Servo Fan System
Mode : HIGH  75%
Dir  : CW
PWM  : 2100
Next : 7s UP
```

## 🚀 Getting Started

### Prerequisites

- Keli uvision5 installed
- ST-Link V2 programmer
- STM32CubeF1 HAL package
- SSD1306 I2C driver library (`ssd1306.h`, `ssd1306_fonts.h`)

### Build & Flash

1. Clone this repository:
   ```bash
   git clone https://github.com/your-username/MotionPulse-STM32-based-PWM-Servo-Control-with-OLED-Feedback.git
   ```
2. Open the project folder in **Keli uvision5**.
3. Build: `Project → Build All`
4. Connect Blue Pill via ST-Link V2.
5. Open STM32 STLink Utility
6. Connect the device -> Program verify -> Select the generated hex file

### Customising Stage Timings or Speeds

Edit the `stages[]` array and `STAGE_HOLD_TIME_MS` in `main.c`:

```c
#define STAGE_HOLD_TIME_MS 10000   // ms per stage (default 10 s)

static const ServoStage stages[STAGE_COUNT] = {
    { 1500, "STOP", "---",  0  },
    { 1700, "LOW",  "CW",  25  },
    { 1900, "MED",  "CW",  50  },
    { 2100, "HIGH", "CW",  75  },
    { 2500, "MAX",  "CW", 100  }
};
```

> ⚠️ Pulse range is 500–2500 µs. Do not exceed `SERVO_MAX_PULSE = 2500` to avoid damaging the servo.
